Based on my research, there isn't a single widely-adopted plugin that does exactly what you're describing (typing @ to trigger a fuzzy file finder inline), but you have several options to achieve this:

Options
1. nvim-cmp + cmp-path with custom trigger

You can configure cmp-path to use @ as a path alias:

lua
{ name = 'path', option = {
  pathMappings = {
    ['@'] = '${folder}/src',  -- maps @ to your src directory
  },
}}
This gives you path completion when typing @/ but it's not a fuzzy finder.

2. Custom nvim-cmp source with trigger character

You could write a custom source that triggers on @ and uses Telescope/fzf under the hood. nvim-cmp supports trigger_characters per source.

3. Keymap-based approach (most practical)

Map @ in insert mode to trigger Telescope's find_files and insert the result:

lua
vim.keymap.set('i', '@', function()
  require('telescope.builtin').find_files({
    attach_mappings = function(_, map)
      map('i', '<CR>', function(prompt_bufnr)
        local selection = require('telescope.actions.state').get_selected_entry()
        require('telescope.actions').close(prompt_bufnr)
        vim.api.nvim_put({ selection.value }, '', true, true)
      end)
      return true
    end
  })
end)
4. blink.cmp - Similar configurability to nvim-cmp, could be extended similarly.
